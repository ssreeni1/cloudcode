# Multi-Provider VPS Support

Branch: main | Mode: HOLD SCOPE

## Context

agentcomputer.ai launched as a managed SaaS for AI agent VMs. cloudcode's competitive strategy: own the **OSS multi-provider self-hosted** position. Beat managed platforms ($200/mo) with $3.49/mo self-hosted on whatever cloud the user already has.

Being Hetzner-only limits addressable market. DigitalOcean is a household name for US devs. Multi-provider removes the single biggest adoption barrier (requiring a Hetzner account).

## Scope

**Ship:**
- `CloudProvider` trait with async methods
- Hetzner refactored into trait impl (zero behavior change for existing users)
- DigitalOcean provider (new)
- VpsState: String IDs, `cloud_provider` field, provisioned metadata
- Config: `[cloud]` section with `CloudKind` enum, migration in init/up only
- Init wizard: cloud provider selection step
- Fix SSH key conflict bug (unique names with fingerprint matching)
- Provider-aware `cloudcode doctor` checks

**Deferred (in TODOS.md):**
- Fly.io — architecturally different (image-based, app-scoped, no cloud-init). Needs own design pass.
- AWS EC2 — complex API (VPC, security groups, IAM). Follow-up PR.
- GCP + Azure — similar complexity to AWS. Follow-up after AWS validates trait handles complex providers.
- Daytona — different integration class (workspace orchestrator, not raw VMs).

## Key Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Architecture | CloudProvider trait (Approach A) | Clean abstraction, community-contributable. ~70 lines more than match dispatch but unlocks extensibility. |
| Naming | "cloud" for infra provider, "provider" stays as AI provider | Avoids naming collision — `provider` already means AI provider everywhere in codebase |
| State IDs | `server_id: String` (was `u64`) | DO uses numeric IDs (fine as String), future providers may use UUIDs |
| Migration trigger | Only in `init`/`up`, NOT in `Config::load()` | Read-only commands (status, doctor) shouldn't trigger file writes. Codex caught this. |
| Destruction provider | From `state.json`, not `config.toml` | If user provisions on Hetzner then changes config to DO, `down` must still use Hetzner |
| SSH key conflict | Unique names (`cloudcode-{fingerprint-prefix}`), fingerprint match on conflict | Current code reuses any key named "cloudcode" and deletes it on down — could delete someone else's key |
| Fly.io | Deferred | Codex outside voice: "Fly is a category error." Image-based, app-scoped, no cloud-init. Different integration class. |
| Config migration | Auto-migrate old format, backup to `config.toml.v1.bak` | Atomic write (tmp+rename). Idempotent. |
| Provisioned metadata | Store server_type + location in state.json | status.rs currently reads from config (lies if config changes after provisioning) |

### What's Provider-Specific vs Generic

```
PROVIDER-SPECIFIC (steps 1-3):       GENERIC (steps 4-10, unchanged):
  - Cloud-init generation              - Wait for SSH
  - SSH key creation (optional)        - Wait for cloud-init
  - Server provisioning                - Verify installation
  - Server destruction                 - Prepare daemon binary
  - Credential validation              - Upload daemon binary
  - Server type listing                - Install daemon service
  - Target triple detection            - Verify daemon
```

## CloudProvider Trait

```rust
#[async_trait]
pub trait CloudProvider: Send + Sync {
    fn name(&self) -> &'static str;

    // Credential validation
    async fn validate_credentials(&self) -> Result<()>;

    // SSH key management (optional — returns None if provider doesn't have key API)
    async fn create_ssh_key(&self, name: &str, public_key: &str) -> Result<Option<String>>;
    async fn delete_ssh_key(&self, id: &str) -> Result<()>;

    // Server lifecycle
    async fn create_server(&self, opts: CreateServerOpts) -> Result<ProvisionedServer>;
    async fn delete_server(&self, id: &str) -> Result<()>;
    async fn get_server(&self, id: &str) -> Result<ServerInfo>;

    // Server types (for TUI picker)
    async fn list_server_types(&self, location: Option<&str>) -> Result<Vec<ServerTypeInfo>>;

    // Architecture detection — takes server_type because providers offer both ARM and x86
    fn target_triple(&self, server_type: &str) -> &'static str;
}

pub struct CreateServerOpts {
    pub name: String,
    pub server_type: String,
    pub image: String,
    pub location: String,
    pub ssh_key_ids: Vec<String>,
    pub user_data: String,
}

pub struct ProvisionedServer {
    pub id: String,
    pub ip: String,
}
```

## Config Changes

```toml
# NEW FORMAT
[cloud]
provider = "hetzner"  # or "digitalocean"

[cloud.hetzner]
api_token = "your-hetzner-token"

[cloud.digitalocean]
api_token = "your-do-token"

# AI provider config unchanged
[claude]
auth_method = "api_key"
api_key = "sk-ant-..."

[codex]
auth_method = "api_key"
api_key = "sk-..."

[telegram]
bot_token = "123456:ABC..."
owner_id = 123456789

[vps]
server_type = "cx23"
location = "nbg1"
image = "ubuntu-24.04"
```

Old format (`[hetzner]` at top level) auto-migrates to new format when user runs `init` or `up`.

## Error Map

```
METHOD/CODEPATH                    | WHAT CAN GO WRONG           | RESCUE ACTION           | USER SEES
-----------------------------------|-----------------------------|-------------------------|------------------
Config migration                   | Corrupt write               | Atomic write (tmp+rename)| Nothing (safe)
Config migration                   | Unknown format              | Warn, proceed defaults  | Warning message
DO validate_credentials            | Invalid token               | Bail with clear message | "DO token invalid"
DO create_server                   | Droplet limit reached       | Catch + explain         | "Increase limit at DO"
DO create_server                   | Payment required (402)      | Catch + explain         | "Add payment method"
DO delete_server                   | API 503                     | Retry 3x with backoff   | Retry, then error
Fly (deferred)                     | N/A                         | N/A                     | N/A
Provider factory                   | Unknown provider string     | Bail + list valid       | "Unknown provider 'x'"
SSH key conflict                   | Key exists, wrong fingerprint| Create with unique name | Nothing (safe)
Status with changed config         | Provider mismatch           | Use state, not config   | Correct info
```

## Implementation Order

1. **`providers/mod.rs`** — CloudProvider trait, CloudKind enum, factory fn
2. **`state.rs`** — String IDs, cloud_provider field, provisioned metadata, backward-compat deserialization
3. **`config.rs`** — CloudConfig section, DOConfig, backward-compat load
4. **`ssh/mod.rs`** — `daemon_socket_path(String)` with length-safe filename (hash/truncate for >20 chars)
5. **`providers/hetzner.rs`** — Refactor hetzner/client.rs into trait impl, unique SSH key names
6. **`providers/digitalocean.rs`** — DO API client + trait impl
7. **`deploy/provision.rs`** — Refactor to use `dyn CloudProvider`
8. **`deploy.rs`** — Move `generate_cloud_init()` to shared location (deploy/cloud_init.rs)
9. **`commands/init.rs`** — Cloud provider selection step, migration trigger
10. **`commands/status.rs`** — Provider from state, provisioned metadata
11. **`commands/down.rs`** — Provider from state for destruction
12. **`commands/up.rs` + `cli.rs`** — Update help text, pass through
13. **`commands/doctor.rs`** — Provider-aware credential checks
14. **Tests** — ~33 new tests across all changed files

## Codex Outside Voice Findings (incorporated)

1. **Fly is a category error** -> Deferred Fly.io

3. **server_id: String insufficient for Fly (needs app+machine)** -> Deferred (String works for Hetzner+DO)

5. **Auto-migrate on load() is bad** -> Migration only in init/up
6. **Trait is Hetzner-shaped** -> Acknowledged, but Hetzner+DO share the same model. Fly deferred.
7. **SSH key conflict bug** -> Fixed with unique names + fingerprint matching
8. **"Only steps 1-3 are provider-specific" is false** -> Expanded scope to include status, doctor, cost estimation
9. **Status reads from config not state** -> Fixed: provisioned metadata in state
10. **"provider" naming collision** -> Renamed to "cloud" for infra provider
